# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you maintain a security vulnerability, please do NOT open a public issue.

### How to report
Please report any security vulnerability by sending an email to **[your-email@domain.com]**. 

### What to expect
* **Acknowledgment:** You will receive a response within 48 hours acknowledging your report.
* **Status Updates:** We will provide weekly updates on the progress of the fix.
* **Disclosure:** Once the vulnerability is patched, a new version will be released, and a security advisory will be published.

### Disclosure Policy
As this is a source-available project with restricted modifications, we ask you to practice **responsible disclosure**. Please give us reasonable time to fix the issue before making any information public.
